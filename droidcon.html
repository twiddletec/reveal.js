What it does

  • measuring test coverage
  • dynamic analyses
  • at any testing phase
  • for C and C++ code
  • for Java

Advantages…

  • Source code based
  • Doesn't require debug build
  • No integration in build files needed, more unwanted
  • Works transparently to the build system
  • With Ant, Maven, ndk-build . . .
  • No shadow directories
  • SDK, NDK, Kernel . . .
  • Works on non-rooted phones/tablets
  • Works on release devices

Example Report

Android applications with SDK

One time setup

ctc.jar as to be added to the $CLASSPATH 
 
#!/bin/bash

if ("x$CLASSPATH" eq "x") then
    export CLASSPATH=/usr/local/lib/ctc.jar:.
else 
    export CLASSPATH=$CLASSPATH:/usr/local/lib/ctc.jar
fi

Android applications with SDK

Compiling

  • Mention the function to trigger data write out in javactc.ini 

#/bin/bash
# Build it with ant, only properties to be added
ant debug -Dbuild.compiler=fi.testwell.ant.ctc \
        -Dbuild.compiler.ctcopts="-i m"
# Install it on your device
adb install -r bin/PACKAGENAME.apk
# Now run it, play around, get coverage data

Android applications with SDK

Data transfer up to report

#!/bin/bash
#Getting report out of device
#Getting MON.txt
adb pull /data/data/PACKAGENAME/MON.txt . 
#Convert date
ctc2dat -i MON.txt && rm MON.txt
# We have a MON.dat
# Proceed as on host
ctcpost MON.dat -p - | ctc2html
firefox CTCHTML/index.html

Android applications with NDK

CTC++ preparations for NDK

  • Copy Host-Target source files to jni/
  • Add entries for targ*.c to jni/Android.mk 
    
LOCAL_SRC_FILES := plasma.c targdata.c targcust.c \
                     targsend.c
  • Mention function name to trigger data writeout 
    EMBED_FUNCTION_NAME = android_main

Android applications with NDK

Building

  • Invoke your build with                                                                           
    ndk-build TARGET_CC="ctc -i m arm-linux-androideabi-gcc"
  • Build then the package with 
    ant debug
  • Install it with 
    adb install -r bin/PACKAGENAME-debug.apk
  • Run your application and test as usual

Coverage on a running Android/Linux kernel

Setup

  • untar ctc7kernel26.tar into kernel top-level directory …
  • … for modules, then 
    make -C /lib/modules/`uname -r`/build
  • You can set a subset of files/directories in .ctc.ini

Coverage on a running Android/Linux kernel

Building

  • Just invoke 
    make CC="ctc arm-linux-androideabi-gcc" …
  • Don't modify CC variable in between, it will rebuild all

Coverage on a running Android/Linux kernel

Preparation on the device

Coverage data dump is done via a character device
  • busybox must be installed on Android
  • At device create the device node with 
    mknod /dev/ctc_coverage c 240 0

Coverage on a running Android/Linux kernel

Data transfer

Transfer the data from target to host and convert on the fly 
adb shell cat /dev/ctc_coverage | ctc2dat

Coverage on a running Android/Linux kernel

Report generating

ctcpost MON.dat | ctc2html
firefox CTCHTML/index.html
